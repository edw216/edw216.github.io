## 알고리즘 성능 분석 기법

알고리즘의 성능 분석 평가 요소에는 2가지 복잡도가 있다.
### 복잡도 분석 방법
--------------
**복잡도란**
+ 알고리즘의 성능을 객관적으로 평가하는 기준이다.


컴퓨터에 상관없이 시간과 공간에 초점을 둔다.
시간복잡도 vs 공간 복잡도

1. **공간 복잡도**
 - 알고리즘이 얼마나 메모리를 적게 쓰는가?
 - 메모리 사용량에 대한 분석결과를 가리킨다.
 - 기억 영역과 파일 공간에 얼마나 필요한가를 평가한 것이다.
 - 
 2. **시간 복잡도**
  - 속도가 얼마나 빠른가? CPU가 얼마나 많은 일을 하는가?
  - 속도에 해당하는 알고리즘의 수행시간 분석결과를 가리킨다.
  - 실행에 필요한 시간을 평가한 것으로 실제 시간은 여러 상황과 조건에 따라 달라질 수 있다.
  - 판단 기준을 연산횟수로 시간 복잡도를 구한다.
  
  ex) 동일조건에서 똑같은 일을 하는 알고리즘 1, 2가 존재 1은 20개의 연산 실행, 2는 100개의 연산 실행  알고리즘1이 알고리즘2에 비하여 실행하는 연산의 수가 적으므로 더 효율적인 알고리즘이다.
  
  

> 연산들의 수행횟수는 입력의 개수 n에 따라 변하고 함수로 나타낸 것을 시간 복잡도 함수라고 하며, T(n)이라고 표기한다.

### 최적의 알고리즘
------------
 - 메모리를 적게 사용하고 속도가 빠른 알고리즘
 - 일반적으로 알고리즘을 평가할 때 메모리의 사용량보다 실행속도에 초점을 둔다. 하지만 공간복잡도도 무시할 수는 없다.
 - 메모리가 부족한 문제보다 메모리에 접근함에 따른 시간이 증가하기 때문에 공간복잡도도 따져봐야한다.
> 시간과 공간을 모두 고려해야한다.
 
최악의 경우, 최상의 경우, 평균적인 경우 3가지 경우로 나누어서 평가한다.

**최상의 경우(best case)**

 - 실행 시간이 가장 적은 경우
 - 아래의 알고리즘과 같이 배열에서 원하는 값을 찾고자 할때 순차적으로 탐색되는 경우에서 배열의 첫번째 요소에서 원하는 값을 찾은 경우를 "최상의 경우"라 하고 시간복잡도 함수정의는 T(n)=1이 된다.

**최악의 경우(worst case)**

 - 실행시간이 가장 오래 걸리는 경우
 - 아래의 알고리즘과 같이 배열에서 원하는 값을 끝에서 찾거나 저장되지 않아서 못찾은 경우를 "최악의 경우"라 하고 시간복잡도 함수정의는 T(n)=n이 된다.

**평균의 경우(Average case)** 

 - 모든 입력을 고려하고 각 입력이 발생하는 확률을 고려한 평균적인 실행 시간의 경우
- 아래의 알고리즘과 같이 중간지점에서 원하는 값을 찾는 경우를 "평균의 경우"라 하고 시간복잡도 함수정의는 상황에 따라 달라진다.

### 빅오(Big-O) 표기법
-------------
**빅오 표기법(Big-O Notation)**

 - 복잡도에서 구한 함수 T(n)에서 실제로 영향력을 끼치는 부분을 큰 O로 감싸 가리켜서 빅-오라 부른다.
 - O는 order의 의미로 차수를 의미한다.

{% raw %}  <img src="https://edw216.github.io/assets/image/Structure(3)/ex7.JPG" alt="">{% endraw %}

위의 알고리즘과 같이 
ex) T(n)=n^2 +n+1 이라는 식이 있다고 하면 n=1000일 때 값은 1,001,001이 되고 n^2은 1,000,000으로 전체의 약 99.9%를 차지한다. 다항식의 경우, 입력의 개수가 큰 경우에 차수가 가장 큰 항이 가장 영향을 크게 미치게 되어서 보통 시간 복잡도 함수에서 최고차 항만을 고려하면 된다.
그래서 가장 큰 차수 n^2가 O(n^2)이 된다.
